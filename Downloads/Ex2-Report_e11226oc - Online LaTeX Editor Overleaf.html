\section{Protocol Desription}

I spent some time thinking through how I wanted to implement my protocol and came to the conclusion that the best way to take in commands and implement features would be to split messages into a command and parameter as advised and then use selection code to execute different code based on the command entered. To follow the requirements I decided to implement 4 commands: Register, Message Broadcast and User List. Together these commands carry out the full implementation from the requirements. I plan on using a large if statement with elif clauses to execute specific code depending on the command input, an else statement will also be used to handle what happens if unrecognised commands are entered. The order of the commands in the if statement shouldn't matter and I plan on putting the if statement inside the onMessage function.

\section{Register:}

\subsection{Description:}
This command allows the user to register a name for themselves on the server, this username needs to be unique to that user.
\subsection{Rationale:}
The command will work by taking in a user input for the username and then checking if there is already a user with that username, if the username isn't already in use it will assign that username to the user. If the username is already in use then there will be a prompt to try another username. I plan on storing the usernames currently in use in a list and then checking this list for new usernames to see if they already exist, if it already exists I will prompt the user to enter a different username.
\subsection{Pseudo Code:}
\begin{verbatim}
currentNames = []
IF (command == "register"):
IF name is in currentNames:
OUTPUT "Sorry this name has already taken, please try again"
ELSE:
socket = name
OUTPUT "Registered as " + name
\end{verbatim}
